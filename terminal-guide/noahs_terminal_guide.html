Terminal & Prompt

iTerm2 + Starship — your daily driver

Action needed: Open iTerm2 (Applications folder) instead of the default Terminal. Then set font: Preferences → Profiles → Text → Font → JetBrainsMono Nerd Font. This makes all icons render correctly.
Must Use
🍎

iTerm2

Why: Apple Terminal doesn't support Nerd Fonts (icons), has no GPU rendering, and can't split panes. iTerm2 is the gold standard Mac terminal.

Split pane: Cmd+D vertical, Cmd+Shift+D horizontal

Active Now
⭐

Starship Prompt

Why: Your prompt now auto-shows git branch, file change counts, language versions, battery %, and time. No extra commands needed.
╭─ ~/hdl/uart main !3
╰─❯ ← your cursor goes here
🔤

JetBrainsMono Nerd Font

Why: Adds 3,000+ icons to a coding font with ligatures. Required for the folder icons, git symbols, and language badges in your prompt to show correctly.

Set in: iTerm2 → Prefs → Profiles → Text → Font

📺

tmux

Why: Run your simulation in one pane, edit code in another, and keep everything in one window. Sessions persist even if you close the terminal.

tmn work → new session · tma work → reattach
Ctrl+B then % → vertical split

Files, Search & Navigation

Modern replacements for ls, cat, grep, and find

🦎 eza — replaces ls

file listing
❯ ll # detailed list: icons + git status + permissions
drwxr-xr-x 📁 src/ +2 untracked
-rw-r--r-- 📄 top.v modified
-rw-r--r-- 📄 testbench.v
 
❯ lt # tree view, 2 levels deep
❯ ltt # tree view, 3 levels deep

🦇 bat — replaces cat

reading files
❯ cat top.v # syntax highlighted Verilog with line numbers
❯ hex rom.bin # hex dump with syntax highlighting (custom function)

🔍 ripgrep (rg) — replaces grep

searching code
❯ rg "module uart" # find in all files recursively
❯ rg "always @" --type v # only search .v files
❯ rg "TODO" -l # just list filenames, not matches

Why over grep: 10-100× faster, respects .gitignore, shows context automatically, handles binary files gracefully.

📂 fd — replaces find

finding files
❯ fd "*.v" # find all Verilog files
❯ fd "testbench" # find files with "testbench" in name
❯ fd ".v" -x vlint # lint every .v file found

⚡ Quick Simulation Commands (Custom Aliases)

simulate Verilog
# vrun — compile and run, see output in terminal
❯ vrun top.v testbench.v
Simulation done. 1024 cycles.
 
# vsim — compile, run, AND open GTKWave automatically
❯ vsim top.v testbench.v
VCD info: dumpfile /tmp/sim.vcd opened.
[GTKWave opens with your waveform]
 
# vlint — lint check without running
❯ vlint top.v
No lint warnings/errors

Note: Your testbench needs $dumpfile("/tmp/sim.vcd"); $dumpvars(0, tb); for vsim to work.

🔬 iverilog — Icarus Verilog (manual usage)

manual compile + run
❯ iverilog -o sim top.v testbench.v # compile
❯ vvp sim # run
❯ gtkwave sim.vcd # open waveform

⚡ Verilator — fast cycle-accurate simulation

verilator
# Lint only (much faster feedback than full simulation)
❯ verilator --lint-only -Wall top.v
 
# Compile to C++ for fast simulation
❯ verilator --cc top.v --exe tb.cpp --build
❯ ./obj_dir/Vtop

Why Verilator: For large designs, it's 100x faster than iverilog. Use iverilog for quick tests, Verilator for serious simulation.

🔧 Yosys — Synthesis

synthesis
# Interactive mode
❯ yosys
yosys> read_verilog top.v
yosys> synth_ice40 -top top
yosys> write_json top.json
 
# Script mode (typical in Makefiles)
❯ yosys -p "synth_ice40 -top top; write_json top.json" top.v

Python Setup

Three separate contexts — never break your system Python again

Why three contexts? macOS Homebrew locks the system Python. Using separate envs prevents conflicts, keeps tools isolated, and means one broken install can't nuke everything else.
🧪

CE Environment (~/.ce-env) — Python 3.12

Your main scientific/hardware Python env. Has numpy, scipy, matplotlib, cocotb, pandas, wavedrom.
❯ ce # activate CE env (alias)
✓ CE Python env active (numpy/scipy/cocotb)
(ce-env) ❯ python3 -c "import numpy; print(numpy.__version__)"

(ce-env) ❯ deactivate # leave the env

CAD & 3D Modeling

Parametric enclosure design with CadQuery — code → STEP/STL

What is CadQuery? A Python library that lets you write parametric 3D models as code. Change a variable, re-run the script, and get a new STEP/STL file. No GUI needed — but you can view results in Onshape (browser) or FreeCAD (desktop).
Core
🧊

CadQuery

Parametric CAD-as-code. Generates STEP, STL, DXF from Python scripts. Built on OpenCascade (OCP) geometry kernel.

pip install cadquery

Viewer
🌐

Onshape

Free browser-based CAD. Import STEP files to view, measure, and edit your models. No install required.

onshape.com — create free account → Import → upload .step

Viewer
🖥️

FreeCAD

Open-source desktop CAD. Full parametric editing, assembly tools, FEM analysis. Get the Apple Silicon build.

brew install --cask freecad

Optional
👁️

CQ-Editor

CadQuery's own GUI — live 3D preview while editing Python scripts. Can be finicky on Apple Silicon.

pip install cq-editor then cq-editor

📦 File Formats — What's What

FormatExtensionUse for
STEP.stepUniversal editable CAD. Open in Onshape, FreeCAD, Fusion 360, SolidWorks. This is your source of truth.
STL.stl3D printing. Load into Cura, PrusaSlicer, BambuStudio. Triangle mesh — not editable.
DXF.dxf2D profiles for laser cutting. CadQuery can export these too.
OBJ.obj3D mesh with materials. Common in rendering/game engines.

🔧 Editing Your Enclosure Parameters

sentinel_enclosure.py — top of file
# --- Overall Enclosure ---
WALL = 3.0 # Wall thickness (mm)
CORNER_RADIUS = 3.0 # External corner rounding
 
# --- Battery Compartment ---
BATT_L = 100.0 # Battery bay length
BATT_W = 60.0 # Battery bay width
BATT_H = 25.0 # Battery bay height
 
# --- Lid Features ---
OLED_W = 30.0 # OLED window width
PWR_BTN_DIA = 12.0 # Power button hole diameter
 
Change any value → save → re-run → new STEP/STL generated
Pro tip: Keep your sentinel_enclosure.py in a git repo. Every time you tweak dimensions and regenerate, commit the script — the STEP/STL files can always be regenerated from it. Track the code, not the binary.
